第一阶段总结

发表于 2021-08-05 更新于 2021-08-06 阅读次数：

关于学习的讨论，敲代码

小黄鸭调试法，每个程序员都要知道的自测法

从一开始JavaSE基础：面向对象 -> JavaSE高级：集合，多线程，注解，反射，IO ->MySQL基础 ->JDBC -> SSM框架
从H5 -> CSS -> JS ->jQuery -> ajax&json -> Vue

Java我看宋红康的，康师傅我觉得讲Java是最好的老师，水平真的高。MySQL我看尚硅谷的。HTML ,CSS,JS，jQuery我看pink老师。SSM框架，ajax我看狂神的。所有老师只有康师傅最牛逼，Java原理讲的太好了。

学着学着，前面的就忘了，就像看书看到200页，前面150页已经忘记了。

可能真的应了那句话，不经常用的东西，时间久了就经忘了。经常用的东西，才会熟悉。

得复习呀，做点笔记啥的。而且你可以一天复习一点，这样可以慢慢补，但是一定要复习。

我想，要么好好做个笔记，要么就是靠平时多用。

科班就是专业对口，比如你的岗位是网络工程师，那么对应科班就是网络工程专业。

个人理解呢，直接学上层当然也能上手，不过这就像专科和本科的区别，懂不懂原理。

平时看的时候也要动手啊，不然基本不会记得多少。代码这种东西还是要多敲，不会的话去看github别人写好简单点的源码，书这种主要还是辅助作用。

重复，没有捷径。

关于JS

我现在承认了，JS确实是重点，因为涉及到了和后端交互，很多东西都是JS写的，差不多90%是JS，10%是HTML。

jQuery就是JS的库，封装了原生的JS代码，就是体现了面向对象的封装性，直接调用方法，方法里面可能是几百行JS实现的功能。write less,do more

我们是从底层学到上层，公司是直接从上层开始使用。感觉底层不学，直接学上层，都可以

关于框架

程序员90%做的业务都在写CRUD（增删改查）
springBoot就是脚手架，springCloud就是微服务
确实，也不是要华的问题，要华当时讲的是底层，那种东西，自己不去多写，都写不出来。而框架就帮你简化了很多，它是不断一层一层去实现各种接口，继承各种类，你直接写一行代码就可以得到，你直接用一个注解就可以实现，不用去管反射是什么原理，根本不用关注内部做了什么，也就是框架帮你把该做的事都写好了。开发现在越来越简单，我现在JavaBean都不写了，直接用注解。
框架框架，千篇一律，一劳永逸。
我JDBC都忘光了，直接配置mybatis
